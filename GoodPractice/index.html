Ejemplos Generales

Siempre es buena practica tener un comentario al principo del archivo que contenga la informacion global de dicho codigo.

Por ejemplo:

/**
 * Author:
 *
 * Contributors:
 *
 * Date:
 * LastUpdate:
 * Version:
 *
 * Name:
 *
 * Description:
 */

Aunque esto depende mas del lenguaje que del codigo en si, es buena practica proporcionar este tipo de informacion

Entre mas expresivo, mejor:

Es mas dificil entender:

...
						r = f1(3,4.5,'m');
						return r;

Que entender:

...
							result = important_equation_select(3,4.5,'m');
							return result;
